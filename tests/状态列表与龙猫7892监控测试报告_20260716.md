# 状态列表与龙猫 7892 监控测试报告

## 一、当前实机状态

用户已把龙猫云代理端口改为 7892。实时扫描结果为：

- Windows 系统代理：开启，Owner 为龙猫云 Lite；
- Windows 系统代理地址：`127.0.0.1:7892`；
- 7892：`lmclientCore` PID 48156；
- 7890：`verge-mihomo` PID 54600；
- Windows 代理联网：正常；
- 当前 Codex 进程环境变量仍指向 `127.0.0.1:7890`；
- 诊断：`MultiOwnerConflict`，浏览器与 Codex 当前走不同客户端。

改为 7892 后，龙猫和 Clash Verge 可以分别监听不同端口，避免 7890 的直接绑定冲突；但 Windows 系统代理、应用环境变量、TUN 和 Helper 仍是独立接管层，不能只靠端口隔离消除全部冲突。

## 二、状态窗口修复

基础状态改为两列表格：

| 项目 | 当前状态 |
|---|---|
| 当前主要代理 | 当前 Owner 摘要 |
| 诊断结果 | 当前诊断标题 |
| Windows 代理 | 已开启或已关闭 |
| Windows 代理来自 | 识别到的 Owner |
| Windows 代理地址 | 例如 `127.0.0.1:7892` |
| 虚拟网卡代理（TUN） | Owner 或未发现 |
| 代理联网 | 正常、异常或未检测 |
| 绕过系统代理 | 正常、异常或未检测 |
| DNS | 正常、异常或未检测 |

诊断说明、发现的问题和建议移到下方独立区域；风险项使用 `1.`、`2.` 编号，并显式使用 Windows `CRLF` 换行，避免截图中多个字段挤在同一行。

## 三、龙猫断连监控修复

默认端口参数由固定 7890 改为自动模式 `0`：

1. 读取当前 Windows 系统代理端口；
2. 确认该端口是否由 `lmclientCore` 监听；
3. 如果 7890 属于其他客户端，则从 7890/7892 中选择实际属于龙猫的端口；
4. 保存实际端口和 `Auto` 模式到当前状态文件；
5. 仍允许显式传入端口，供固定端口诊断使用。
