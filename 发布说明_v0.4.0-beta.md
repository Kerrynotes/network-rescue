# 断网急救 v0.4.0-beta 发布说明

## 本版定位

v0.4.0-beta 不再让用户选择“恢复直连”还是“保留某个代理客户端”，而是统一为：

> 一键退出全部代理并恢复普通网络。

清场成功后，如需代理，请只手动打开一个客户端。

## 主要变化

- 托盘一级菜单收敛为四项，不再显示“更多工具”、目标客户端选择和低频诊断记录入口；
- 新增公开模式 `EmergencyDirect`；
- `StopAllClients` 映射到 `EmergencyDirect`；
- `PrepareSwitch` 和 `StopOtherClients` 停止执行；
- 新增 FlClash 同源运行时冲突、动态 IPC 失效和 Clash Verge 孤儿核心诊断；
- 先正常退出 UI，等待最多 5 秒，再精确清理服务、核心、TUN、路由、DNS 和本地代理配置；
- 只有普通权限清理后仍有受保护残留时才请求 UAC；
- 系统代理、代理残留和普通网络分别验收，失败时只显示“部分完成”。
- 2026-07-16 修复 Helper 服务路径被 `sc.exe` 错误拆分而导致安装返回 1639 的问题。
- 2026-07-16 修复状态文件短暂占用触发 .NET JIT 错误窗口并终止托盘监控的问题。
- 2026-07-16 新增 Clash Party 路径级识别，修复其 `mihomo.exe`被误认为 Clash Verge 的问题。
- 2026-07-16 网络状态窗口增加冲突详情、“退出全部代理并恢复普通网络”和“暂不处理”按钮。
- 2026-07-16 修复非主客户端退出时误关主代理的 Windows 系统代理：端口归属、PID 和代理联网复核通过后自动恢复；手动关闭和主代理退出不触发。
- 2026-07-16 修复 Windows 代理关闭但 Codex 仍通过环境变量联网时误报“代理接管正常”的问题。
- 2026-07-16 网络状态改为“项目 / 当前状态”两列表格，风险和建议按编号分区显示。
- 2026-07-16 龙猫云断连监控改为自动识别 7890/7892；龙猫使用 7892、Clash 占用 7890 时不再误报龙猫断连。
- 2026-07-16 删除右键托盘菜单中的“诊断报告与记录”，诊断数据仍作为内部排查能力保留。
- 2026-07-16 后台日志和事件增加容量轮转，龙猫云断连记录增加 180 天和 5000 条双重上限，避免长期运行后无限占用磁盘。

## 安装和权限

普通安装不需要管理员权限。

以下情况需要管理员权限：

- 停止受保护代理服务；
- 结束受保护核心；
- 清理高权限 TUN、路由或机器级代理设置；
- 安装或卸载断网急救 Helper。

可选 Helper 安装时确认一次 UAC，之后执行白名单内高权限动作不再弹 UAC，但每次清场仍会显示产品内确认。

如果旧压缩包曾显示“创建 Helper 服务失败：1639”，请换用重新打包后的文件，再运行“安装高权限Helper_仅需一次UAC.bat”。

## 升级说明

直接解压新版本并运行“安装断网急救.bat”即可升级。升级不会覆盖已有日志和备份。
