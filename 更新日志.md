# 断网急救更新日志

## v0.4.2-beta 通用代理连接监控（2026-07-18）

### 监控范围修正

- 断连监控不再只针对龙猫云，而是跟随当前实际接管网络的系统代理、应用代理、TUN 或普通直连路径。
- 确认代理线路故障时记录当前 Owner、代理地址和失败原因；恢复后记录持续时间。
- 新增 `monitor_data\代理连接记录.csv`，保留最近 180 天、最多 5000 条；普通用户通过“查看代理连接记录.bat”打开。
- 删除龙猫云专用断连监控脚本和入口，避免用户误以为其他代理客户端没有被监控。

## v0.4.1-beta 启动入口收敛（2026-07-18）

### 产品收敛

- 新增“启动断网急救.bat”，用于托盘菜单手动退出后的重新启动；它调用已经安装的主程序，不会重复执行安装。
- 移除“启动只读后台监控”这一容易误解的名称，避免用户误以为退出后必须重新安装。
- 普通用户入口收敛为安装、启动、查看连接记录和卸载；低频的扫描、手动清场、手动停止监控和手动启停 Helper 批处理文件不再随分享包提供。

## v0.4.0-beta 托盘菜单精简（2026-07-16）

### 产品收敛

- 删除右键托盘菜单中低频的“诊断报告与记录”入口，一级菜单从五项收敛为四项。
- 诊断数据和运行日志仍在后台自动保留，仅供故障排查，不再作为日常操作暴露给用户。
- 后台日志增加容量轮转：普通日志单文件最大 2 MB，结构化事件单文件最大 5 MB，均只保留 3 份归档。
- 龙猫云断连 CSV 改为保留最近 180 天且最多 5000 条；当前状态文件仍每次覆盖，不会累积历史快照。

### 验证

- 托盘自检改为校验四个保留入口，并显式拒绝诊断记录入口回归。
- 新增监控日志、修复动作日志、Helper 日志的轮转自检，以及龙猫云断连记录的时间和条数边界测试。

## v0.4.0-beta 端口与状态窗口修复（2026-07-16）

### 修复

- 网络状态窗口改为“项目 / 当前状态”两列表格，系统代理地址独立显示；诊断说明、风险项和建议改为分段编号列表，修复状态文字挤在同一段中难以阅读的问题。
- 龙猫云独立断连监控不再默认写死 7890，改为自动识别 7890/7892，并优先选择当前 Windows 系统代理实际指向且由 `lmclientCore` 监听的端口。
- 7890 被 Clash Verge 占用、龙猫云改用 7892 时，监控会选择 7892，不再误报“端口 7890 被其他程序占用”导致龙猫已断连。

### 验证

- 状态窗口模型自检确认生成 9 项状态列表、Windows 代理地址独立成行、风险项使用编号和 Windows 换行符。
- 龙猫云监控自检覆盖“7890 属于 Clash、7892 属于龙猫”的自动选择场景。
- 当前实机自动采样选择 7892，识别 `lmclientCore` PID 48156，代理联网成功 2/3。

## v0.4.0-beta 系统代理误关修复（2026-07-16）

### 修复

- 修复 Clash Party 等非主客户端退出时关闭共享的 Windows 系统代理，导致龙猫云核心和 7890 仍正常但浏览器转为直连的问题。
- 自动急救新增高置信度恢复分支：确认退出的是非主客户端、原系统代理 Owner 和同一 PID 仍监听原端口、无 TUN、无代理守护且代理联网复核成功后，自动重新开启当前用户的 Windows 系统代理。
- 用户单独手动关闭系统代理、主代理核心退出、端口监听者变化或代理联网失败时不会自动重开。
- 修复 Windows 系统代理关闭、无 TUN、仅 Codex/终端通过环境变量继续走代理时仍显示“代理接管正常”的误判；现在显示 `ApplicationPathSplit` 和 Windows 未被代理接管。

### 验证

- 新增“Clash Party 退出、龙猫云继续监听 7890”的候选恢复回归测试。
- 新增手动关闭系统代理、主代理退出两条禁止自动恢复的边界测试。

## v0.4.0-beta 安装修复（2026-07-16）

### 修复

- 修复安装高权限 Helper 时，`sc.exe` 错误拆分 `C:\Program Files\...` 服务路径和 `--sid` 参数，导致“创建 Helper 服务失败：1639”的问题。
- Helper 服务改由 Windows `Win32_Service` 接口创建，不再依赖 `sc.exe create` 的命令行引号转换。
- UAC 被取消或未能启动时仍保持 Helper 未安装状态，不影响普通监控和当前代理客户端运行。
- 修复托盘操作恰逢日志或状态文件被其他程序短暂占用时，文件异常从点击回调冒泡并触发 .NET JIT 错误窗口、导致托盘监控退出的问题。
- 日志、事件和状态文件写入增加有限重试；所有托盘点击及气泡通知动作增加统一异常边界，单次操作失败时只提示重试，后台监控继续运行。
- 新增 Clash Party 适配器，识别 `Clash Party.exe`及其 `resources\sidecar\mihomo.exe`核心。
- Clash Verge 不再仅凭通用进程名 `mihomo.exe`认领核心；`verge-mihomo.exe`继续按专属名称识别。
- 通用 Mihomo 核心改为结合可执行文件路径判断；路径未知或属于其他客户端时显示未知并拒绝结束。
- TUN 名称同时匹配多个 Mihomo 客户端时，结合当前实际运行的 UI/Core 归属；仍有多个候选时保持未知，不抢先归属列表中的第一个客户端。
- “网络状态”由只显示“确定”的信息框改为可操作窗口：发现风险时列出具体冲突，并提供“退出全部代理并恢复普通网络”和“暂不处理”。
- 状态窗口中的 `Healthy`、`NotTested` 等内部枚举改为“正常”“未检测”等中文新手文案；进入清场前继续保留完整对象确认。

### 验证

- Windows PowerShell 5.1 语法解析通过。
- Helper 编译、自检和客户端参数白名单测试通过。
- 新增独占文件锁和托盘点击异常回归测试，确认异常被隔离且不会进入 .NET JIT 对话框。
- 实机确认 Clash Party 的 `mihomo.exe`监听7891、7892和9090均正确归属 Clash Party，Clash Verge 不再显示为运行或孤儿核心。
- 新增状态窗口模型测试，覆盖无冲突不显示修复按钮、存在冲突显示详情和一键解决入口。
- 当前会话中的 UAC 被取消，未执行高权限实装；修复包需重新运行“安装高权限Helper_仅需一次UAC.bat”完成最终安装验证。

## v0.4.0-beta（2026-07-15）

### 产品收敛

- 面向新手只保留“一键退出全部代理并恢复普通网络”一个核心动作。
- 托盘一级菜单收敛为网络状态、一键清场、低风险自动急救、诊断记录和退出五项。
- 删除目标客户端选择和安全切换入口；`PrepareSwitch`、`StopOtherClients` 停止执行。
- `StopAllClients` 兼容映射到新的 `EmergencyDirect`。

### 新增

- 新增 FlClash 同源运行时、共享 Helper 端口和动态 IPC 端口适配器字段。
- 新增 `SharedRuntimeConflict`、`ClientIpcBroken`、`OrphanCore` 和清场后 `ProxyResidue` 验收。
- 新增 `EmergencyCleanupStarted`、`EmergencyCleanupCompleted`、`EmergencyCleanupPartial`、`SharedRuntimeConflictDetected`、`ClientIpcBrokenDetected` 和 `OrphanCoreReleased` 事件。
- 新增另一台电脑 12460 失效、13145 恢复的脱敏样本回放测试。

### 清场流程

- 先关闭系统代理，再向全部已知 UI 发送正常退出请求并统一等待最多 5 秒。
- 停止服务后，对残留进程重新核对 PID、名称、路径、启动时间和客户端白名单再结束。
- 不调用机场客户端自己的共享 `/stop` 接口，避免同源客户端交叉误杀。
- 定向清理已知 TUN、`198.18.*` 路由、代理 DNS、WinHTTP 和本地代理环境变量。
- 普通权限先执行；只有最终仍有受保护残留时才调用 Helper 或请求一次 UAC。
- 系统代理、代理残留和普通网络分别验收，UAC 取消或任何最终项失败时显示“部分完成”。

### 修复

- 修复端口 Owner 在状态切换时缺少 `UiRunning` 字段导致扫描中断的问题。
- 修复清场后只根据退出码误报完成的风险，改为读取结构化结果并复核三项状态。
- Helper 的 TUN 清理增加代理 DNS 和全部已知 `198.18.*` 路由处理。
- 修复自定义目录沙箱卸载错误删除正式安装开机快捷方式的问题；卸载前必须确认快捷方式确实指向当前安装目录。

### 验证

- PowerShell 5.1 解析、扫描器自检、修复引擎自检、托盘自检和脱敏样本回放通过。
- 本机真实清场通过：系统代理关闭、已知代理残留清空、直连探针成功。
- UAC 取消场景正确显示“部分完成”；确认 UAC 后高权限清场完整通过。
- 清场后用户手动只打开龙猫云，龙猫重新接管 7890 和系统代理，满足“清场后只打开一个客户端”的流程。

## v0.3.3-beta（2026-07-15）

### 修复

- 修复 Helper 安装失败时只显示“没有成功安装或启动”，无法判断 UAC 取消、服务创建失败还是通信失败的问题。
- Helper 安装流程新增结构化结果文件，记录失败阶段、原因和服务状态；托盘提示会直接显示这些信息。
- 区分“普通网络恢复已经成功”和“后续高权限步骤尚未执行”，避免用户误以为前面的恢复操作也失败了。
- 安装 Helper 时把原始用户 SID 传给高权限安装进程，避免使用其他管理员凭据时把服务权限授予错误账号。
- 服务安装完成后必须通过命名管道通信验证，不能只以“服务已启动”作为成功标准。

### 验证

- 当前机器确认 Helper 服务尚未安装；`Program Files` 中只有上次中断留下的文件。
- 无管理员权限测试可以生成“检查权限：当前进程没有管理员权限”的结构化结果。
- 新增 UAC 取消、安装阶段错误和残留文件识别回归测试。

## v0.3.1-beta（2026-07-15）

### 修复

- 修复切换网络、重新启动监控或代理端口暂时失效时，托盘菜单显示后无法点击的问题。
- 完整 Owner 扫描、手动重新检查和诊断报告生成全部改为后台执行，不再占用托盘 UI 线程。
- 失效系统代理达到3次确认后，普通网络直连探测改为后台执行；探测期间托盘菜单保持可操作。
- 轻量端口检查不再调用耗时的 CIM 查询，改用本机 IP Helper 监听快照。实机稳定状态下，单次检查由约522～1436毫秒降至约1～39毫秒。
- 启动监控时不再等待第一次完整扫描结束；托盘先进入“正在检查”，结果完成后再更新状态。

### 验证

- 新增慢网络 UI 心跳回归测试：后台探测故意延迟1.2～1.5秒时，托盘消息循环仍持续响应。
- 新增阻塞式端口查询回归门禁，防止后续重新把 CIM 端口查询放回 UI 线程。

## v0.3.0-beta（2026-07-14）

### 新增

- 统一生成 `NetworkSnapshot`、`PathHealth` 和 `Diagnosis`，保留旧扫描字段兼容。
- 新增多目标联网探测：分别检查普通网络、当前系统代理、应用环境变量代理和 DNS。
- 新增 `HealthyDirect`、`HealthySingleOwner`、`MixedModeSameOwner`、`StaleSystemProxy`、`MultiOwnerConflict`、`TunResidual`、`ApplicationPathSplit`、`ProxyPathDegraded` 和 `LocalNetworkFailure` 等诊断状态。
- 新增端口 Owner 证据，包括客户端、核心进程、PID、启动时间、会话和监听端口。
- 新增安全切换向导：选择目标客户端、完整退出其他客户端、等待用户手动连接并在5分钟内自动复核。
- 新增上次成功目标记忆，只作为下次默认选项，不强制锁定客户端。
- 新增 Codex/终端应用代理同步与清理；写入前重新确认本地端口仍在监听。
- 新增线路断连、端口 Owner、TUN、应用路径分裂和安全切换结构化事件。
- 未安装 Helper 时，可选择一次 UAC 安装 Helper，或仅本次请求 UAC。

### 调整

- 托盘一级菜单继续保持5项；“断网时自动恢复普通上网”更名为“失效系统代理时自动恢复”。
- 同一客户端同时开启系统代理和 TUN只显示混合模式信息，不判为多 Owner 冲突。
- 机场线路或节点失效不再触发自动关闭代理：30秒静默标记，60秒记录事件，持续3分钟后提醒用户更新节点或恢复普通网络。
- “恢复普通网络”会列出准备清理的本地代理环境变量，并提醒重启 Codex、终端等已运行应用。
- 龙猫云断连监控改为复用通用三目标探测，旧批处理入口和日志格式继续保留。
- 高权限 Helper 仍使用固定操作和 `ClientId` 白名单；结束进程前增加名称、PID和启动时间复核。

### 安全边界

- 后台自动操作仍只关闭连续确认无人监听的 Windows 系统代理开关。
- 后台不会自动结束进程、停止服务、清理 TUN、修改 DNS、清理环境变量或切换节点。
- 停止服务、结束核心和清理 TUN始终需要产品内确认；安装 Helper 后只是免去后续 UAC。
- 无法确认归属或协议能力的端口、路由、DNS和环境变量保持只读。
