## 五、测试结论与限制

通过发布门槛：

- 同源客户端冲突识别；
- 动态 IPC 失效识别；
- 孤儿核心识别；
- Helper-only 边界；
- UAC 取消边界；
- 本机真实清场；
- 手动只打开一个客户端后重新接管。

发布沙箱测试还发现并修复了一个安装器隔离问题：自定义测试目录卸载时不再删除指向正式安装目录的开机快捷方式，状态检查也只报告当前安装目录对应的快捷方式。

另一台电脑没有重新制造冲突，结论来自既有日志和脱敏会话样本。事故发生时未启用进程审计，因此不能把“某个二次开发客户端确定发送了停止请求”写成已证实事实。

## 六、2026-07-16 Clash Party 归属修复补充测试

实机启动 Clash Party 后确认：

- UI：`Clash Party.exe`；
- Core：安装目录 `resources\sidecar\mihomo.exe`；
- Core 监听：7891、7892、9090；
- 配置根目录：`%APPDATA%\mihomo-party`；
- 修改前：通用 `mihomo.exe`被 Clash Verge 适配器抢先匹配，并错误触发 `OrphanCore`；
- 修改后：UI 与 Core 均归属 Clash Party，三个端口均显示 `clientId=clash_party`；
- 修改后：Clash Verge 显示未运行，不再认领 Clash Party 核心；
- 其他路径或没有路径证据的 `mihomo.exe`保持未知。

新增回归测试覆盖：Clash Party 路径匹配、其他 Mihomo 路径拒绝、Clash Verge 不抢占通用核心、停止进程前专属路径复核。Windows PowerShell 5.1 解析、扫描器自检、修复引擎自检和 Helper 自检均通过。

状态窗口同步增加冲突解决入口。模型测试确认：无冲突时只显示“关闭”；存在冲突时显示冲突详情、“退出全部代理并恢复普通网络”和“暂不处理”；选择修复后继续进入原有清场确认，不会直接结束客户端。
